0. Agent 生成 PPT + 口播文本

输入一句自然语言需求，或上传参考资料。Web 端可配置 OpenAI 兼容 LLM 接口，也可以给每个参考文件指定类型和用途。

支持类型建议：`outline` 大纲、`material` 素材、`data` 数据、`brand` 品牌规范、`source_ppt` 参考 PPT。

如果未配置 LLM，系统会用本地规则生成可编辑初稿；配置后会调用 OpenAI 兼容 `/v1/chat/completions` 接口。

如果返回 `HTTP 403`，优先检查 API Key、模型权限、Base URL 是否填到正确网关，以及是否需要在“自定义请求头 JSON”里补内部网关要求的 Header。

1. 提取声纹

建议上传 10 到 60 秒、只有一个人说话、背景噪音尽量少的音频。一般直接用 `cpu + 覆盖已存在声纹` 即可。

`auto` 会自动选择设备。没有可用 GPU 时，推荐直接选 `cpu`，兼容性最好。

勾选后，同名声纹会被重新生成；不勾选时，如果同名文件已存在会报错。

2. PDF 分割图片

默认配置适合大多数 PPT/PDF。想要更清晰的图片可以提高渲染倍率，但文件体积也会更大。

`render_scale`：推荐 `1.5` 到 `3.0`，越大越清晰，也越占空间。

`image_format`：默认 `png` 清晰度更稳；如果想减小文件体积可改成 `jpg`。

`start_page / end_page`：可只导出一部分页面。`end_page=0` 表示一直导到最后一页。

3. 图片 + scripts.json + 声纹合成视频

先准备好图片序列和 `scripts.json`，再选择声纹合成视频。大多数情况下保留默认参数就能直接出结果。

可直接选择内置声纹，也可以先在上面的“提取声纹”模块里生成新的声纹。

`speaker_seed / text_seed`：用于控制随机性。想稳定复现同样结果时，保持不变即可。

`temperature / top_p / top_k`：控制发音生成的随机程度。默认值通常最稳，不建议大幅修改。

`max_new_token`：单段文本特别长时才需要调大，普通口播一般不用动。

`refine_prompt`：控制口语化、停顿等风格；如果你不确定含义，保留默认即可。

`video_fps`：静态 PPT 配音视频通常 `25` 就够用。

`video_crf`：数值越小画质越高、体积越大。推荐 `18` 到 `28`，默认 `23` 比较均衡。

`video_preset`：越慢压缩率通常越好，但导出更慢。默认 `medium` 最均衡。

`audio_bitrate`：常用 `96k` 或 `128k`，默认 `128k` 即可。

`关闭文本润色`：如果你希望尽量按原文念，可以勾选；否则建议关闭，让模型自动优化口播节奏。

`保留 TTS 中间音频 / 保留中间文件`：只在排查问题时建议开启，平时关闭更省空间。
