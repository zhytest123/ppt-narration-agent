二、启动前准备

1. 启动命令

cd web_system
./run.sh

启动成功后，浏览器打开 `http://127.0.0.1:8000`。

2. 你需要准备什么

  • 如果要提取声纹：准备一个单人说话音频
  • 如果要切图：准备一个 PDF 文件
  • 如果要合成视频：准备图片序列、`scripts.json` 和一个声纹
推荐准备方式： 声纹音频尽量选 10 到 60 秒、只有一个人说话、没有明显背景音乐的片段。这样提取结果更稳定。

三、模块 1：提取声纹

使用场景

当你想让后续 TTS 模拟某个人的说话音色时，第一步就是上传该人的参考音频并提取声纹。

操作步骤

  1. 在“提取声纹”模块上传音频文件。
  2. 可选填写“声纹名称”，不填则默认使用音频文件名。
  3. 设备建议选 `cpu`，兼容性最好。
  4. 保持“覆盖已存在声纹”勾选，避免同名时报错。
  5. 点击“提取声纹”，等待状态区返回结果。
图 2：声纹提取界面示意
提取声纹
左侧是表单，右侧是返回结果。提取成功后，新声纹会自动进入后续视频模块的下拉框。
参数 推荐值 说明
`device` `cpu` 最稳妥，适合大多数机器。
`force` 开启 同名声纹会覆盖更新，省去手动清理旧文件。

四、模块 2：PDF 分割图片

使用场景

当你的内容来源是 PPT 导出的 PDF 时，先把每一页拆成图片，再去准备每页文案，是最直接的做法。

操作步骤

  1. 上传 PDF 文件。
  2. 保留默认参数，或修改渲染倍率、页码范围。
  3. 点击“分割并下载图片 ZIP”。
  4. 浏览器会自动开始下载压缩包。
  5. 解压后即可得到按页命名的图片序列。
图 3：PDF 切图结果示意
默认文件名类似 `page_001.png`。建议后续 `scripts.json` 直接沿用这些文件名，最不容易出错。
参数 推荐值 说明
`render_scale` `2.0` 清晰度和体积比较均衡。
`image_format` `png` 更稳，不容易糊；如果特别在意体积，可考虑 `jpg`。
`start_page / end_page` `1 / 0` `0` 表示一直到最后一页。

五、模块 3：图片 + 文稿 + 声纹合成视频

你需要准备三样东西

  • 一组图片，例如 `page_001.png`、`page_002.png`
  • 一个 `scripts.json` 文件
  • 一个可选的声纹，来自内置声纹或你自己提取的新声纹

操作步骤

  1. 先在下拉框中选择声纹。
  2. 上传 `scripts.json`。
  3. 上传图片序列。
  4. 可选填写输出文件名，例如 `demo.mp4`。
  5. 保持默认 TTS 和视频参数，点击“开始合成视频”。
  6. 等待任务状态变成 `done`，再点击“下载/打开视频”。
图 4：视频合成界面示意
开始合成视频
状态区会持续轮询任务状态。合成完成后，会出现最终视频链接。

`scripts.json` 示例

{
  "pages": [
    { "image": "page_001.png", "script": "第1页口播文案" },
    { "image": "page_002.png", "script": "第2页口播文案" }
  ]
}
最重要的规则： `scripts.json` 里的 `image` 文件名，必须和你上传的图片文件名完全一致。只要名字对不上，视频任务就会失败。
参数组 推荐值 怎么理解
TTS 设备 `cpu` 最稳，尤其适合打包给别人使用。
`speaker_seed / text_seed` `42 / 42` 保持不变，方便稳定复现。
`temperature / top_p / top_k` `0.3 / 0.7 / 20` 默认值通常最稳，不建议大幅修改。
`video_fps / video_crf` `25 / 23` 静态口播视频常用配置。

六、常见问题

1. 提取声纹失败

  • 检查音频是否损坏
  • 检查是否多人说话
  • 检查音频格式是否受支持
  • 同名时建议保持 `force` 开启

2. PDF 切图失败

  • 确认上传的真的是 PDF
  • 确认页码范围是否合理
  • `render_scale` 填写不要为 0 或负数

3. 视频合成失败

  • `scripts.json` 是否是合法 JSON
  • 图片文件名是否完全匹配
  • 声纹是否存在
  • 输出文件名建议以 `.mp4` 结尾

4. 页面改了但看不到

  • 强制刷新浏览器
  • 重新打开页面
  • 重启 `./run.sh`
如果你是第一次用，建议严格按这个顺序：先提声纹 → 再切图 → 最后合成视频。这样出错点最少。
